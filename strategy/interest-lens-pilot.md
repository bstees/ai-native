# Interest Lens Pilot

## Purpose

Use `Interest Lens` as the first real repository to test both the backlog
curation process and the sharing model for native-AI assets.

## Why This Repo First

`Interest Lens` gives us a live environment where we can validate whether the
curation process produces useful work and whether shared assets are adoptable in
practice.

## Pilot Objectives

### Objective 1: Test Curated Backlog Flow

- Feed curated concept records into `Interest Lens`
- Review whether generated backlog items are specific, relevant, and actionable
- Measure whether the curation output helps prioritize work more effectively

### Objective 2: Test Cross-Repo Asset Sharing

- Introduce a small number of reusable assets into `Interest Lens`
- Observe what is easy versus awkward to adopt
- Learn whether assets should be synced, copied, or generated in place

## Pilot Scope

Start small:

- one source registry draft
- one concept ontology draft
- one first-pass backlog report
- one or two shared assets consumed by `Interest Lens`

## Questions To Answer

- Which curated concepts actually matter in a real repo?
- What evidence is strong enough to justify backlog creation?
- Which asset types are easiest to share centrally?
- Where does prompt-based generation outperform static shared files?

## Success Signals

- Curated backlog items are understandable without extra interpretation
- `Interest Lens` can adopt at least one shared asset with minimal friction
- We learn which asset categories belong in a central source-of-truth repo
- We identify the first automation worth building next

## Near-Term Work

1. Define the source registry and concept record formats in this repo.
2. Produce the first curation output targeted at `Interest Lens`.
3. Choose one native-AI asset type to pilot across repos.
4. Capture adoption findings and feed them back into the sharing strategy.
