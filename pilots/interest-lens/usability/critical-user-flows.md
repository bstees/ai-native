# Interest Lens Critical User Flows

## Purpose

Define the critical flows that should be exercised directly during usability
validation.

## Initial Flows

### 1. Enter debt information

- Goal:
  a user can input debt details without confusion or hidden controls
- Verify:
  fields are reachable, labels are clear, actions remain visible across view
  states

### 2. Review analysis results

- Goal:
  a user can understand the result hierarchy and locate the next recommended
  action
- Verify:
  primary insights, totals, and next steps are visually obvious

### 3. Compare payoff or savings options

- Goal:
  a user can compare options without losing context or actionability
- Verify:
  comparisons remain readable, controls remain reachable, and state changes are
  understandable

### 4. Recover from errors or incomplete inputs

- Goal:
  a user can understand how to continue when data is missing or invalid
- Verify:
  error states are readable and recovery actions are obvious
