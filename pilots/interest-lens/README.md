# Interest Lens Pilot

## Purpose

Use `Interest Lens` as the first consumer repository for `AI Native`.

## What This Pilot Tests

### 1. Signal Quality

Do the concepts and decisions coming out of `AI Native` reflect common threads
that matter in a real production-style repo?

### 2. Asset Adoption

Can `Interest Lens` consume shared prompts, rules, workflows, or other assets
with low friction?

### 3. Feedback Loop

Does adoption in `Interest Lens` reveal which assets should stay centralized and
which should instead be generated in place?

## Early Scope

- consume the first approved shared assets
- review what lands well versus what feels premature
- feed adoption findings back into `AI Native` decisions
